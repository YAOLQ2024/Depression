# 快速开始：构建和合并自定义知识库

这是一个简化的快速开始指南，帮助你快速上手构建自己的知识库。

## 🚀 三步走

### 步骤1: PDF转TXT

```bash
# 方法1: 使用PaddleOCR（推荐）
pip install paddlepaddle opencv-python paddleocr
python scripts/pdf2txt.py /path/to/your/pdf/folder

# 转换后的TXT文件会保存在 res/ 目录
```

### 步骤2: 构建你的知识库

```bash
cd rag

# 方法1: 使用自动化脚本（推荐）
python build_my_knowledge_base.py /path/to/your/txt/files

# 方法2: 手动构建
# 1. 将TXT文件复制到 rag/src/data/txt/ 目录
# 2. 运行构建脚本
cd src
python data_processing.py
```

### 步骤3: 合并知识库

```bash
cd rag
python merge_vector_db.py
```

合并后的知识库会保存在 `rag/merged_knowledge_base/vector_db/`

### 步骤4: 使用合并后的知识库

编辑 `rag/src/config/config.py`，修改向量库路径：

```python
vector_db_dir = os.path.join(cloud_vector_db_dir, 'merged_knowledge_base', 'vector_db')
```

然后运行你的web应用：

```bash
streamlit run web_internlm2.py --server.address=0.0.0.0 --server.port 7860
```

## ⚠️ 注意事项

1. **确保官方知识库已下载**：合并前需要先下载官方知识库
   ```bash
   git clone https://code.openxlab.org.cn/Anooyman/EmoLLMRAGTXT.git
   ```

2. **数据格式**：
   - TXT文件必须是UTF-8编码
   - JSON文件格式参考教程文档

3. **路径配置**：如果遇到路径问题，请检查 `rag/src/config/config.py` 中的配置
