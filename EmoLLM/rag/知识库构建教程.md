## 步骤1: PDF/文本转TXT

### 方法一：使用PaddleOCR（推荐，支持扫描版PDF）

#### 1.1 安装依赖

```bash
pip install paddlepaddle
pip install opencv-python
pip install paddleocr
```

**注意**：如果无法使用 `pip install paddleocr` 安装，可以：
- 从 [PyPI](https://pypi.org/project/paddleocr/#files) 下载对应的 whl 文件进行安装
- 或使用项目根目录下的 `careyou/faiss_gpu-1.7.2-cp310-cp310-manylinux_2_17_x86_64.manylinux2014_x86_64.whl`（如果适用）

#### 1.2 使用脚本转换

项目提供了 `scripts/pdf2txt.py` 脚本用于PDF转TXT：

```bash
# 使用方法：python pdf2txt.py [PDF存放的文件夹路径]
python scripts/pdf2txt.py /path/to/your/pdf/folder
```

**脚本功能**：
- 自动扫描指定文件夹及其子文件夹中的所有PDF文件
- 使用PaddleOCR进行OCR识别（支持中文）
- 将提取的文本保存为TXT文件到 `res/` 目录

**输出**：
- 转换后的TXT文件会保存在 `res/` 目录下
- 文件名与原始PDF文件名相同，扩展名为 `.txt`

### 方法二：使用API转换（需要API Key）

如果你有 doc2x API key，可以使用 `rag/pdf2md/pdf2md.py`：

```python
from rag.pdf2md.pdf2md import PDF2MD

api_key = "your-api-key-here"
converter = PDF2MD(api_key)
converter.convert("your_file.pdf", to="md")  # 会同时生成txt和md文件
```

### 方法三：直接使用TXT文件

如果你已经有TXT格式的文本文件，可以直接使用，跳过转换步骤。

## 步骤2: 准备知识库数据

### 2.1 创建数据目录结构

在 `rag/src/data/` 目录下创建以下结构：

```
rag/src/data/
├── txt/          # 存放TXT文本文件
│   ├── book1.txt
│   ├── book2.txt
│   └── ...
└── json/          # 存放JSON格式的QA对（可选）
    ├── qa1.json
    ├── qa2.json
    └── ...
```

### 2.2 准备TXT文件

将转换好的TXT文件复制到 `rag/src/data/txt/` 目录：

```bash
# 示例：将转换好的TXT文件复制到数据目录
cp res/*.txt rag/src/data/txt/
```

**注意事项**：
- 确保TXT文件使用UTF-8编码
- 可以放入多个TXT文件，系统会自动处理所有文件
- 建议对文本进行初步清洗，去除无关内容（如目录、页码等）

### 2.3 准备JSON格式的QA对（可选）

如果你有结构化的QA对数据，可以将其放入 `rag/src/data/json/` 目录。

**JSON格式示例**：

```json
[
    {
        "conversation": [
            {
                "input": "用户问题1",
                "output": "回答1"
            },
            {
                "input": "用户问题2",
                "output": "回答2"
            }
        ]
    }
]
```

或者每行一个JSON对象（JSONL格式）：

```jsonl
{"conversation": [{"input": "问题1", "output": "回答1"}]}
{"conversation": [{"input": "问题2", "output": "回答2"}]}
```

## 步骤3: 构建自己的向量库

### 3.1 配置参数

编辑 `rag/src/config/config.py`，确认以下配置：

```python
# 数据目录
data_dir = os.path.join(base_dir, 'data')
doc_dir = os.path.join(data_dir, 'txt')    # TXT文件目录
qa_dir = os.path.join(data_dir, 'json')    # JSON文件目录

# 文本切分参数
chunk_size = 1000      # 每个文本块的大小
chunk_overlap = 100    # 文本块之间的重叠大小

# 向量库保存路径（自定义知识库）
vector_db_dir = os.path.join(base_dir, 'my_knowledge_base', 'vector_db')
```

### 3.2 构建向量库

运行以下命令构建向量库：

```bash
cd rag/src
python data_processing.py
```

**构建过程**：
1. 加载embedding模型（BAAI/bge-small-zh-v1.5）
2. 读取 `txt/` 目录下的所有TXT文件并切分
3. 读取 `json/` 目录下的所有JSON文件并处理
4. 对所有文档进行embedding
5. 构建FAISS向量索引
6. 保存到 `vector_db_dir` 指定的目录

**输出**：
- 向量库文件会保存在配置的 `vector_db_dir` 目录下
- 包含 `index.faiss` 和 `index.pkl` 两个文件

## 步骤4: 合并知识库

### 4.1 使用合并脚本

我们提供了专门的合并脚本 `rag/merge_vector_db.py` 来合并你的知识库和官方知识库。

#### 4.1.1 配置合并参数

编辑 `rag/merge_vector_db.py`，设置以下路径：

```python
# 官方知识库路径
official_vector_db_dir = "rag/EmoLLMRAGTXT/vector_db"

# 你的知识库路径
my_vector_db_dir = "rag/my_knowledge_base/vector_db"

# 合并后的知识库保存路径
merged_vector_db_dir = "rag/merged_knowledge_base/vector_db"
```

#### 4.1.2 运行合并脚本

```bash
cd rag
python merge_vector_db.py
```

**合并过程**：
1. 加载embedding模型
2. 加载官方知识库
3. 加载你的知识库
4. 合并两个向量库
5. 保存合并后的向量库

### 4.2 手动合并（高级用法）

如果你想更精细地控制合并过程，可以参考以下代码：

```python
from rag.src.data_processing import Data_process
from langchain_community.vectorstores import FAISS
import os

# 初始化数据处理对象
dp = Data_process()
emb_model = dp.load_embedding_model()

# 加载官方知识库
official_db = FAISS.load_local(
    "rag/EmoLLMRAGTXT/vector_db",
    emb_model,
    allow_dangerous_deserialization=True
)

# 加载你的知识库
my_db = FAISS.load_local(
    "rag/my_knowledge_base/vector_db",
    emb_model,
    allow_dangerous_deserialization=True
)

# 合并向量库
merged_db = official_db  # 以官方库为基础
merged_db.merge_from(my_db)  # 合并你的知识库

# 保存合并后的向量库
merged_vector_db_dir = "rag/merged_knowledge_base/vector_db"
os.makedirs(merged_vector_db_dir, exist_ok=True)
merged_db.save_local(merged_vector_db_dir)
```

## 📝 常见问题

### Q1: PDF转换后文本质量不好怎么办？

**A**: 
- 对于扫描版PDF，PaddleOCR效果较好
- 对于文字版PDF，可以尝试使用其他工具如 `pdfplumber` 或 `PyPDF2`
- 转换后建议手动检查和清洗文本

### Q2: 如何调整文本切分参数？

**A**: 编辑 `rag/src/config/config.py`：
- `chunk_size`: 增大可以包含更多上下文，但可能影响检索精度
- `chunk_overlap`: 增大可以保持上下文连贯性，但会增加存储空间

### Q3: 合并后的向量库很大，加载慢怎么办？

**A**: 
- 考虑使用更高效的索引类型（如 `IndexIVFFlat`）
- 减少 `chunk_size` 和 `chunk_overlap` 参数
- 对知识库进行筛选，只保留高质量内容

### Q4: 如何更新知识库？

**A**: 
- 添加新的TXT文件到 `txt/` 目录
- 重新运行 `data_processing.py` 构建新的向量库
- 使用合并脚本将新知识库与现有知识库合并

### Q5: 向量库合并后，如何验证是否成功？

**A**: 
- 检查合并后的向量库文件大小是否增加
- 使用测试查询验证能否检索到新旧知识库的内容
- 查看日志确认合并的文档数量

## 🔧 高级技巧

### 技巧1: 批量处理PDF

创建一个批处理脚本：

```bash
#!/bin/bash
# batch_convert.sh
for pdf in /path/to/pdfs/*.pdf; do
    python scripts/pdf2txt.py "$(dirname "$pdf")"
done
```

### 技巧2: 文本预处理

在构建向量库前，可以对TXT文件进行预处理：

```python
import re

def clean_text(text):
    # 移除多余空白
    text = re.sub(r'\s+', ' ', text)
    # 移除特殊字符（根据需要调整）
    text = re.sub(r'[^\w\s\u4e00-\u9fff]', '', text)
    return text.strip()
```

### 技巧3: 增量更新知识库

如果只想添加新内容而不重建整个知识库：

```python
# 加载现有向量库
existing_db = FAISS.load_local(vector_db_dir, emb_model, allow_dangerous_deserialization=True)

# 处理新文档
new_docs = dp.split_document("new_file.txt")

# 添加到现有向量库
existing_db.add_documents(new_docs)
existing_db.save_local(vector_db_dir)
```
