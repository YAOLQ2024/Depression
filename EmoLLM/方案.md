# 抑郁症知识库构建与合并完整方案

本文档针对你的具体需求：**将20个关于抑郁症预防、诊断、调控的PDF文件加入到官方知识库**，提供两套完整的方案。

## 📋 方案概览

### 方案一：官方标准流程（推荐用于高质量知识库）
- **优点**：数据质量高，经过LLM筛选和QA对构造，检索精度高
- **缺点**：流程复杂，需要API调用，耗时较长
- **适用场景**：构建专业、高质量的知识库

### 方案二：简化流程（推荐用于快速添加知识库）
- **优点**：流程简单，速度快，无需API调用
- **缺点**：可能存在重复数据，需要手动清洗
- **适用场景**：快速添加知识库，对质量要求不严格

### 步骤5：将清洗后的QA对转换为向量库可用格式

#### 5.1 转换jsonl为JSON格式

清洗后的QA对是jsonl格式（每行一个JSON对象），需要转换为向量库可以使用的格式。

**创建转换脚本** `scripts/qa_generation/convert_jsonl_to_json.py`：

```python
import os
import json
import glob

def convert_jsonl_to_json(jsonl_dir, output_dir):
    """
    将jsonl文件转换为JSON格式，供向量库使用
    """
    os.makedirs(output_dir, exist_ok=True)
    
    # 查找所有jsonl文件
    jsonl_files = glob.glob(os.path.join(jsonl_dir, '*.jsonl'))
    
    for jsonl_file in jsonl_files:
        qa_list = []
        
        # 读取jsonl文件
        with open(jsonl_file, 'r', encoding='utf-8') as f:
            for line in f:
                line = line.strip()
                if line:
                    try:
                        qa_data = json.loads(line)
                        # 转换为向量库需要的格式
                        # 每个QA对作为一个conversation
                        qa_list.append({
                            "conversation": [
                                {
                                    "input": qa_data.get("question", ""),
                                    "output": qa_data.get("answer", "")
                                }
                            ]
                        })
                    except json.JSONDecodeError as e:
                        print(f"Error parsing line in {jsonl_file}: {e}")
                        continue
        
        # 保存为JSON文件
        if qa_list:
            base_name = os.path.splitext(os.path.basename(jsonl_file))[0]
            output_file = os.path.join(output_dir, f"{base_name}.json")
            with open(output_file, 'w', encoding='utf-8') as f:
                json.dump(qa_list, f, ensure_ascii=False, indent=2)
            print(f"Converted {jsonl_file} -> {output_file} ({len(qa_list)} QA pairs)")

if __name__ == '__main__':
    # 清洗后的QA对目录
    cleaned_dir = './data/cleaned'
    # 输出目录（向量库使用的目录）
    output_dir = '../../rag/data/json'
    
    convert_jsonl_to_json(cleaned_dir, output_dir)
    print("Conversion completed!")
```

**运行转换脚本**（二选一）：
```bash
cd scripts/qa_generation
# python convert_jsonl_to_json.py 会转换目录下全部 .jsonl。
python convert_jsonl_to_json.py

# 支持 --pattern 指定 glob 模式（例：--pattern "*SDS*.jsonl"），只处理匹配文件。同时加入 --input-dir、--output-dir 可调输入输出目录。
python convert_jsonl_to_json.py --pattern "*SDS*.jsonl"
```

**输出**：转换后的JSON文件会保存在 `rag/data/json/` 目录

### 问题2：简化流程没有QA对提取和清洗，会不会有性能问题？

**详细分析**：

#### 2.1 关于检索速度

**结论：影响很小**

**原因**：
- FAISS的检索速度主要取决于**向量数量**，而不是数据质量
- 即使有重复数据，FAISS的检索算法（基于向量相似度）速度仍然很快
- 20个PDF文件转换后的向量数量通常在几千到几万级别，对检索速度影响很小

**数据对比**：
- 官方知识库：可能有几十万到上百万个向量
- 你的20个PDF：估计几千到几万个向量
- 合并后：总向量数增加很少，检索速度影响可忽略

#### 2.2 关于检索精度

**结论：可能有一定影响，但可以通过其他方式优化**

**潜在问题**：
1. **重复内容**：
   - 如果TXT文件中有重复段落，会被切分成多个相似的向量块
   - 检索时可能返回多个相似结果，但FAISS的rerank机制可以缓解

2. **无关内容**：
   - 如果TXT中包含目录、页码等无关内容，可能被检索到
   - 但可以通过调整chunk_size和chunk_overlap参数优化

3. **上下文丢失**：
   - 直接切分TXT可能破坏语义完整性
   - QA对方式保持了问答的完整性，语义更清晰

**优化建议**：
1. **手动清洗TXT文件**（推荐）：
   - 删除目录、页码、页眉页脚
   - 保留核心内容

2. **调整切分参数**：
   ```python
   # 在 rag/src/config/config.py 中调整
   chunk_size = 1000      # 增大可以包含更多上下文
   chunk_overlap = 100    # 增大可以保持连贯性
   ```

3. **使用rerank机制**：
   - 在 `rag/src/pipeline.py` 中启用rerank
   - 可以过滤掉不相关的结果

#### 2.3 实际影响评估

**对于你的场景（20个抑郁症PDF）**：

| 方面 | 影响程度 | 说明 |
|------|---------|------|
| 检索速度 | ⭐ 很小 | 向量数量增加很少，速度影响可忽略 |
| 检索精度 | ⭐⭐ 中等 | 可能有重复，但rerank可以缓解 |
| 数据质量 | ⭐⭐ 中等 | 取决于TXT文件质量 |
| 开发成本 | ⭐⭐⭐ 低 | 流程简单，无需API调用 |

**建议**：
- 如果你的PDF质量好，TXT转换清晰，**简化流程完全可以满足需求**
- 如果对检索精度要求很高，或者PDF质量一般，建议使用**官方流程**
