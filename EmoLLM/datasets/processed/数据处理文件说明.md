## 单轮对话数据处理

### 1. `process_single_turn_conversation_construction.py`

**功能**：将单轮对话数据从 `prompt`/`completion` 格式转换为 `conversation` 格式

**处理的数据文件**：
- `single_turn_dataset_1.json`
- `single_turn_dataset_2.json`

**输入格式**：
```json
[
    {
        "prompt": "用户输入的问题",
        "completion": "模型的回答"
    }
]
```

**输出格式**：
```json
[
    {
        "conversation": [
            {
                "system": "你是心理健康助手EmoLLM...",
                "input": "用户输入的问题",
                "output": "模型的回答"
            }
        ]
    }
]
```

**使用方法**：
```bash
cd datasets/processed
python process_single_turn_conversation_construction.py
```

**输出文件**：

`processed_single_turn_dataset_1.json`
`processed_single_turn_dataset_2.json`

## 多轮对话数据处理

### 4. `process_multi_turn_conversation.py`

**功能**：为多轮对话数据添加 `system` 字段（只给第一个 turn 添加）

**处理的数据文件**：
- `multi_turn_dataset_1.json`
- `multi_turn_dataset_2.json`

**输入格式**：
```json
[
    {
        "conversation": [
            {"input": "第一轮用户输入", "output": "第一轮模型输出"},
            {"input": "第二轮用户输入", "output": "第二轮模型输出"}
        ]
    }
]
```

**输出格式**：
```json
[
    {
        "conversation": [
            {
                "input": "第一轮用户输入",
                "output": "第一轮模型输出",
                "system": "你是心理健康助手EmoLLM..."
            },
            {
                "input": "第二轮用户输入",
                "output": "第二轮模型输出"
            }
        ]
    }
]
```

**使用方法**：
```bash
cd datasets/processed
# 修改脚本中的 file_name 变量选择要处理的文件
python process_multi_turn_conversation.py
```

**输出文件**：`processed_multi_turn_dataset_1.json` 或 `processed_multi_turn_dataset_2.json`

**注意**：只给第一个 turn 添加 `system` 字段，这是推荐的做法，因为：
- 符合实际对话场景（system prompt 通常在对话开始时设置一次）
- 数据更紧凑，减少冗余
- 符合主流微调框架的预期

---

### 5. `process.py`

**功能**：为已有 `conversation` 格式的数据添加 `system` 字段（原地修改）

**处理的数据文件**：
- `data.json`（或其他已有 `conversation` 格式但缺少 `system` 的数据）

**输入格式**：
```json
[
    {
        "conversation": [
            {"input": "...", "output": "..."},
            {"input": "...", "output": "..."}
        ]
    }
]
```

**输出格式**：
```json
[
    {
        "conversation": [
            {"input": "...", "output": "...", "system": "..."},
            {"input": "...", "output": "..."}
        ]
    }
]
```

**使用方法**：
```bash
cd datasets/processed
# 修改脚本中的 file_name 变量选择要处理的文件
python process.py
```

**输出文件**：`processed_data.json`

**与 `process_multi_turn_conversation.py` 的区别**：
- `process.py`：原地修改数据，更简单直接
- `process_multi_turn_conversation.py`：重新构建数据结构，更安全但更慢

## 数据合并与分割

### 8. `process_merge.py`

**功能**：合并当前目录下所有 JSON 文件的数据

**处理的数据文件**：
- 当前目录（`datasets/processed/`）下所有 `.json` 文件

**使用方法**：
```bash
cd datasets/processed
python process_merge.py
```

**输出文件**：`combined_data.json`

**注意**：
- 会合并所有 JSON 文件的数据到一个列表中
- 如果文件不是列表格式，会作为单个元素添加

**功能**：将数据集划分为训练集和测试集

**处理的数据文件**：
- 通过修改脚本中的 `input_jsonl_path` 变量指定

**使用方法**：

- `train_emo.jsonl`：训练集
- `test_emo.jsonl`：测试集

## 数据去重

### 11. `deduplicate.py`

**功能**：使用绝对匹配（MD5）和相似性匹配（SimHash）对数据进行去重

**处理的数据文件**：
- 通过命令行参数指定目录下的所有 `.json` 文件

**去重方法**：
1. **绝对匹配**：使用 MD5 哈希检测完全相同的记录
2. **相似性匹配**：使用 SimHash 检测相似度超过阈值的记录（默认阈值 0.8）

**使用方法**：
```bash
cd datasets/processed

# 使用默认参数（从 ../ 目录读取，输出到 ./dedup）
python deduplicate.py

# 指定输入目录和阈值
python deduplicate.py --input_dir ../ --threshold 0.85

# 开启详细日志
python deduplicate.py --input_dir ../ --log_items

# 查看帮助
python deduplicate.py --help
```

**命令行参数**：
- `--input_dir`：输入目录路径（默认：`../`，即 datasets/ 目录）
- `--output_dir`：输出目录路径（默认：`./dedup`）
- `--threshold`：相似度阈值（默认：0.8）
- `--log_items`：是否记录每条数据的日志（默认：False）
- `--file_pattern`：文件匹配模式（默认：`*.json`）

**输出文件**：
- `dedup_<原文件名>.json`：去重后的数据
- `dup_<原文件名>.json`：重复的数据

**依赖包**：
```bash
pip install loguru datasketch simhash numpy
```

### 功能重复性分析

#### 1. 单轮对话格式转换（有重复）

**重复的文件**：
- `process_single_turn_conversation_construction.py`
- `process_self_cognition.py`

- `Book_QA_process_Step_2.py`

**区别**：
- **输入格式不同**：
  - `process_single_turn_conversation_construction.py`：`prompt`/`completion`

  - `Book_QA_process_Step_2.py`：`prompt`/`completion`（书籍 QA 专用）

- **system prompt 略有不同**：
  - `process_single_turn_conversation_construction.py`：较短的 system prompt
  - `process_self_cognition.py`：较长的 system prompt
  - `ruozhiba_raw_data_process.py`：较长的 system prompt
  - `Book_QA_process_Step_2.py`：书籍 QA 专用的 system prompt

**建议**：可以统一为一个通用脚本，通过参数指定输入格式和 system prompt。

- `process_multi_turn_conversation.py`

**建议**：
- 需要数据验证时使用 `process_multi_turn_conversation.py`
- 快速处理时使用 `process.py`

## 数据格式说明

### 最终用于微调的数据格式

所有处理后的数据都应该符合以下格式：

```json
[
    {
        "conversation": [
            {
                "system": "你是心理健康助手EmoLLM...",
                "input": "用户输入",
                "output": "模型输出"
            }
        ]
    }
]
```

**多轮对话格式**（只有第一个 turn 有 system）：
```json
[
    {
        "conversation": [
            {
                "system": "你是心理健康助手EmoLLM...",
                "input": "第一轮用户输入",
                "output": "第一轮模型输出"
            },
            {
                "input": "第二轮用户输入",
                "output": "第二轮模型输出"
            }
        ]
    }
]
```

**注意**：
- `system` 字段只在第一个 turn 中出现，这是推荐的做法
- 微调框架（如 XTuner）会在训练时通过 `template_map_fn` 将数据转换为模型特定的格式
- 数据还需要经过 tokenization 等处理才能用于训练

## 注意事项

1. **路径问题**：所有脚本都使用相对路径，需要在 `datasets/processed/` 目录下运行
2. **文件修改**：运行前需要检查脚本中的 `file_name` 变量，确保指向正确的文件
3. **数据备份**：处理大量数据前建议先备份原始数据
4. **依赖安装**：`deduplicate.py` 需要额外安装依赖包
5. **数据格式**：确保输入数据格式符合脚本要求，否则可能处理失败
