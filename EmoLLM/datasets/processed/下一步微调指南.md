### 步骤 1：验证数据文件

首先确认数据文件已正确生成：

```bash
cd datasets/processed

# 检查数据文件
python -c "
import json
import os

for category in ['general', 'roleplay']:
    category_dir = f'final_datasets/{category}'
    train_file = f'{category_dir}/train.json'
    test_file = f'{category_dir}/test.json'
    
    if os.path.exists(train_file) and os.path.exists(test_file):
        with open(train_file, 'r', encoding='utf-8') as f:
            train_data = json.load(f)
        with open(test_file, 'r', encoding='utf-8') as f:
            test_data = json.load(f)
        
        total = len(train_data) + len(test_data)
        print(f'{category.upper()} 类:')
        print(f'  训练集: {len(train_data)} 条 ({len(train_data)/total*100:.1f}%)')
        print(f'  测试集: {len(test_data)} 条 ({len(test_data)/total*100:.1f}%)')
        print(f'  总计: {total} 条')
        print(f'  绝对路径: {os.path.abspath(category_dir)}')
        print()
    else:
        print(f'❌ {category.upper()} 类数据文件不存在')
"
```

### 步骤 6：模型转换与合并

#### 6.1 将 PTH 模型转换为 HuggingFace 格式

```bash
cd xtuner_config/

# 创建输出目录
mkdir hf

# 转换模型（替换为你的实际路径）
export MKL_SERVICE_FORCE_INTEL=1
xtuner convert pth_to_hf \
    internlm2_7b_chat_qlora_e3.py \
    ./work_dirs/internlm_chat_7b_qlora_oasst1_e3_copy/epoch_3.pth \
    ./hf
```

#### 6.2 合并 Adapter 到基础模型

```bash
cd xtuner_config/

# 合并模型（替换为你的实际路径）
xtuner convert merge \
    /path/to/base/model \
    ./hf \
    ./merged \
    --max-shard-size 2GB
```

## 📝 重要提示

### 1. 数据格式要求

确保你的数据格式符合 Xtuner 的要求：

```json
[
  {
    "conversation": [
      {
        "system": "你是心理健康助手EmoLLM...",
        "input": "用户输入",
        "output": "模型输出"
      }
    ]
  }
]
```

**你的数据已经是这个格式** ✅

### 2. 选择训练数据

- **General 类**：通用心理健康对话数据
- **Role-play 类**：角色扮演类数据（如 aiwei、mother、scientist）

**建议**：
- 如果只训练通用模型，使用 `general/train.json`
- 如果需要角色扮演能力，使用 `roleplay/train.json`
- 也可以合并两类数据一起训练

### 3. 显存优化

如果显存不足，可以：
- 减小 `batch_size`
- 增加 `accumulative_counts`（梯度累积）
- 使用 4-bit 量化（QLoRA）
- 减小 `max_length`

### 4. 训练监控

训练过程中关注：
- **Loss 下降趋势**：应该逐渐下降
- **评估输出**：检查模型生成质量
- **检查点保存**：确保定期保存

## ❓ 常见问题

### Q1: 如何选择使用哪个配置文件？

**A**: 根据你的基础模型选择：
- InternLM2 → `internlm2_7b_chat_qlora_e3.py`
- Qwen → `qwen_7b_chat_qlora_e3.py`
- Llama3 → `llama3_8b_instruct_qlora_alpaca_e3_M.py`

### Q2: 训练需要多长时间？

**A**: 取决于：
- 数据量大小
- 模型大小
- GPU 性能
- 训练轮数（epochs）

一般 7B 模型在单卡 A100 上，3 个 epoch 可能需要几小时到一天。

### Q3: 如何判断训练是否成功？

**A**: 检查：
- Loss 是否持续下降
- 评估输出的质量是否提升
- 测试时模型回答是否更符合预期
