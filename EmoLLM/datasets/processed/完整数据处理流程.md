## 阶段 3：数据去重

### 目标
使用绝对匹配（MD5）和相似性匹配（SimHash）去除重复和相似数据。

### 步骤 3.1：对 General 类数据去重

```bash
cd datasets/processed

python deduplicate.py \
    --file combined_general.json \
    --output_dir ./dedup_general \
    --threshold 0.8
```

**参数说明**：
- `--file`：指定要处理的文件（只处理这一个文件）
- `--output_dir`：输出目录
- `--threshold`：相似度阈值（0.8 表示相似度超过 80% 视为重复）

**输出**：
- `dedup_general/dedup_combined_general.json`：去重后的数据
- `dedup_general/dup_combined_general.json`：重复的数据

**性能说明**：
- 优化后的代码使用 SimhashIndex，速度大幅提升
- 对于 4.8 万条数据，从原来的 O(n²) 优化到 O(n×k)，k 为桶平均大小
- 预计处理时间：几分钟到十几分钟（取决于数据量和硬件）

### 步骤 3.3：查看去重统计

```bash
cd datasets/processed

# 查看去重结果
python -c "
import json
import os

for category in ['general', 'roleplay']:
    dedup_file = f'dedup_{category}/dedup_combined_{category}.json'
    dup_file = f'dedup_{category}/dup_combined_{category}.json'
    
    if os.path.exists(dedup_file):
        with open(dedup_file, 'r', encoding='utf-8') as f:
            dedup_data = json.load(f)
        with open(dup_file, 'r', encoding='utf-8') as f:
            dup_data = json.load(f)
        
        total = len(dedup_data) + len(dup_data)
        dup_rate = len(dup_data) / total * 100 if total > 0 else 0
        
        print(f'{category.upper()} 类:')
        print(f'  原始数据量: {total}')
        print(f'  去重后数据量: {len(dedup_data)}')
        print(f'  重复数据量: {len(dup_data)}')
        print(f'  去重率: {dup_rate:.2f}%')
        print()
"
```

## 数据格式说明

### 最终数据格式

所有处理后的数据都应符合以下格式：

#### 单轮对话格式

```json
[
    {
        "conversation": [
            {
                "system": "你是心理健康助手EmoLLM...",
                "input": "用户输入",
                "output": "模型输出"
            }
        ]
    }
]
```

#### 多轮对话格式（只有第一个 turn 有 system）

```json
[
    {
        "conversation": [
            {
                "system": "你是心理健康助手EmoLLM...",
                "input": "第一轮用户输入",
                "output": "第一轮模型输出"
            },
            {
                "input": "第二轮用户输入",
                "output": "第二轮模型输出"
            }
        ]
    }
]
```

**重要**：
- `system` 字段只在第一个 turn 中出现，这是推荐的做法
- 符合实际对话场景（system prompt 通常在对话开始时设置一次）
- 数据更紧凑，减少冗余
- 符合主流微调框架的预期

## 注意事项

### 1. 数据分类的重要性

- **General** 和 **Role-play** 数据必须分开处理
- 两类数据的 `system` prompt 不同
- 分开训练可以更好地控制模型行为
- 可以根据需要选择使用通用模型还是角色扮演模型

### 2. 文件大小

- 合并后的 JSON 文件可能达到 100MB+，这是正常的
- 如果内存充足（16GB+），可以直接使用 JSON 格式
- 如果想减小文件，可以：
  - 使用 JSONL 格式（减小 30-50%）
  - 去掉缩进（减小 20-30%）
  - 使用 gzip 压缩（减小到 20-30MB）

### 3. 路径问题

- 所有脚本都使用相对路径
- 必须在 `datasets/processed/` 目录下运行脚本
- 确保文件路径正确

### 4. 数据备份

- 处理大量数据前建议先备份原始数据
- 去重后的数据也要备份
- 可以使用版本控制（git）管理数据

### 5. 性能优化

- `deduplicate.py` 已优化，使用 SimhashIndex 加速
- 对于大数据集，去重可能需要几分钟到十几分钟
- 如果处理时间过长，可以：
  - 降低相似度阈值（但可能丢失重要数据）
  - 分批处理数据
  - 使用更强大的硬件

### 6. 数据验证

- 每个阶段完成后都要验证数据格式
- 检查数据量是否符合预期
- 检查去重率是否合理（通常 5-20% 是正常的）
